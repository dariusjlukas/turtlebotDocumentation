Turtlebot3 Quickstart

So, the turtlebot is set up, you have a computer set up to work with it, you are on the same network as the turtlebot, the turtlebot is powered on, your computer is turned on, the wifi is working, you are sure the turtlebot is on the same wifi as you are, the turtlebot it not currently beeping at you, and it is not sitting somewhere high off the ground. The basic steps to make the turtlebot drive are as follows:

  1. Launch the roscore application on the master computer (likely your computer). To do this, run:
    $ roscore
    in a terminal. It should complete looking something like:
    Do NOT type this into your terminal!
    Press Ctrl-C to interrupt
    Done checking log file disk usage. Usage is <1GB.
    started roslaunch server http://192.168.10.129:38411/
    ros_comm version 1.12.14
    SUMMARY
    ========
    PARAMETERS
    * /rosdistro: kinetic
    * /rosversion: 1.12.14
    NODES
    auto-starting new master
    process[master]: started with pid [8814]
    ROS_MASTER_URI=http://192.168.10.129:11311/
    setting /run_id to 09283c5e-9d01-11e9-a5ce-2c6e8579d489
    process[rosout-1]: started with pid [8827]
    started core service [/rosout]
  2. SSH into the turtlebot (or hookup a monitor, mouse, and keyboard) and run the command:
    $ roslaunch turtlebot3_bringup turtlebot3_robot.launch
    This will use roslaunch to launch the turtlebot3_bringup package using the turtlebot3_robot.launch launchfile. To ssh into a turtlebot you can use the command:
    $ ssh pi@turtlebot{turtlebot number}
    replacing {turtlebot number} with the number of the turtlebot you are ssh'ing into. If prompted for a password type: "turtlebot". If the host cannot be resolved, replace "turtlebot#" with the ip address of the turtlebot, which is written on each turtlebot and looks like: 192.168.10.{something}.

    If the roslaunch turtlebot3_bringup turtlebot3_robot.launch command completes successfully, you should see output like:
    Do NOT type this into your terminal!
    ...
    [INFO] [1562098245.600566]: Setup publisher on /tf [tf/tfMessage]
    [INFO] [1562098245.623450]: Note: subscribe buffer size is 1024 bytes
    [INFO] [1562098245.625336]: Setup subscriber on cmd_vel [geometry_msgs/Twist]
    [INFO] [1562098245.652826]: Setup subscriber on sound [turtlebot3_msgs/Sound]

  3. Start the remote turtlebot package on your computer. To do this, run this command on your computer:
    $ roslaunch turtlebot3_bringup turtlebot3_remote.launch
    You should see output similar to:
    Do NOT type this into your terminal!
    Checking log directory for disk usage. This may take awhile.
    Press Ctrl-C to interrupt
    Done checking log file disk usage. Usage is <1GB.
    started roslaunch server http://192.168.10.129:40973/

  4. You have completed all of the basic setup steps to launch the turtlebot's ROS nodes, so if you type:
    $ rostopic list
    you should see a variety of topics. Rviz can be used to easily visualize the data:
    $ rosrun rviz rviz -d `rospack find turtlebot3_description`/rviz/model.rviz
    To try driving the robot around, you can use:
    $ roslaunch turtlebot3_teleop turtlebot3_teleop_key.launch
    You can also cause the turtlebot to move by directly publishing to the /cmd_vel topic. To publish to a ROS topic from the command line, use:
    $ rostopic pub {topic name} {topic message type} {topic message}
    For example, to make the turtlebot go forwards you could use:
    $ rostopic pub /cmd_vel geometry_msgs/Twist "linear:
        x: 1.0
        y: 0.0
        z: 0.0
    angular:
        x: 0.0
        y: 0.0
        z: 0.0"
    To stop it, run:
    $ rostopic pub /cmd_vel geometry_msgs/Twist "linear:
        x: 0.0
        y: 0.0
        z: 0.0
    angular:
        x: 0.0
        y: 0.0
        z: 0.0"
    To list the topics you can publish to, run:
    $ rostopic list
    If you want to figure out the message format for any given topic, simply his [tab] after typing "rostopic pub {topic name}". For example, a quick way to publish to the motor topic would be:
    $ rostopic pub /cmd_vel [tab] [tab]


    To try out more examples with the turtlebot3, consult the e-Manual , which is where I got these commands from.
